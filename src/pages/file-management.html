Gerenciamento de Arquivos

No geral, os princípios por trás do gerenciamento de arquivos de um sistema operacional são os mesmos, o que muda, dentre cada um deles é a forma de manuseá-los. A principal distinção pode ser notada ao manipular arquivos por meio da linha de comando, em que o Windows e Linux apresentam diferenças, ao mesmo tempo que algumas semelhanças. A seguir serão esclarecidos alguns exemplos.

Linux

O sistema de arquivos funciona por meio de pastas, desde sua raiz. Todos os dados do sistema são distribuídos através de diretórios e assim segue. A origem de tudo fica por meio do diretório /. Isto é, o sistema parte disso. Para dar um vislumbre em toda a estrutura primária de pastas, basta ir em alguma aplicação de linha de comando e operar o comando ls /. Com isso, será possível visualizar os diretórios alicerce em que todo o sistema está montado.

Windows

Já, no Windows, o esquema é um pouco diferente. Todo o sistema se divide em Unidades de Disco, e a partir delas é que um sistema de diretórios é montado. A forma mais rápida de se visualizar uma Unidade é simplesmente entrar em seu explorador de arquivo e ver o nome de seu Disco Local. Ele provavelmente vai ser o C:. Essa é uma unidade de disco. Para exemplificar, podemos demonstrar o caminho de pastas destinadas ao uso típico do usuário, que se encontra em C:/Users/SeuUsuario/. Na linha de comando do Linux, você sabe que chegou na sua pasta “lar” quando aparece ~, ao invés do caminho.
