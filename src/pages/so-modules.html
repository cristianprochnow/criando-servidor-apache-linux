Módulos de um SO

Antes de qualquer outra coisa, temos que definir o porquê que o termo “módulo” pode ser empregado nesse contexto também.

Aqui, cada “módulo” nada mais é que um pedacinho do sistema operacional, focado e muito bom em realizar uma determinada tarefa. Cada módulo desempenha um conjunto de tarefas em um determinado contexto. E, assim, o Sistema Operacional se torna um grande amontoado de vários e vários módulos, que juntos proporcionam a interface humano-máquina que nos é tão útil nos dias atuais.

Exemplos de Módulos

Para exemplificar, podemos elencar abaixo alguns módulos básicos existentes em um Sistema Operacional.

Módulo de Gerenciamento de Arquivos

É o módulo do sistema operacional que fica responsável por manipular a criação e existência de arquivos como um todo. Seja para criar pastas, seja para criar arquivos, editá-los, apagá-los e tantas outras operações. Esse módulo é quem te proporciona mexer com o conteúdo de um arquivo.

Módulo de Gerenciamento de Tarefas

Módulo responsável por controlar os aplicativos e programas que estão rodando ao longo do tempo no seu PC. Este módulo é quem é sempre chamado naquele momento clássico de desespero em que um programa para de responder e não há esperança de recuperação para esse. Nessa situação, o módulo gerenciador de tarefas é completamente capaz de olhar a situação e então dar um jeito nisso tudo, seja fechando forçadamente um programa, seja finalizando tarefas por si só.

Módulo Gerenciador de Recursos

E, no último exemplo, temos o gerenciador de recursos, que é a parte do sistema responsável por administrar, da melhor forma possível, os recursos em hardware disponíveis para o sistema operacional. É por meio deste módulo que o uso de processador, memória, processamento gráfico e assim por diante são usados de forma racional, e aplicados corretamente no que realmente é necessário.
